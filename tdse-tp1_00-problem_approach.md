# Solución de COMA Electronics

La solución 'Intelligent Parking Management System' desarrollada por COMA Electronics consiste en una infraestructura integrada para la administración automatizada de playas de estacionamiento. Esta plataforma centraliza sus operaciones a través de un servidor dedicado al sistema general. A nivel de campo, la arquitectura despliega máquinas dispensadoras para el control de entrada y terminales de lectura para la salida, ambas ofreciendo diversas configuraciones operativas. El ecosistema comercial se completa integrando computadoras de cobro y estaciones de pago automático para procesar las transacciones de los usuarios antes de que retiren sus vehículos.

# Implementación de 'Parking Ticket Dispenser Machine (Entry)'

Respecto a la implementación específica de la máquina dispensadora de tickets en la entrada, el software se concibe bajo una estructura de aplicación estrictamente modular, la cual está dividida en tres etapas secuenciales: escrutar, procesar y actuar. Estos módulos de código logran comunicarse y sincronizar sus tareas de forma exclusiva a través del intercambio de mensajes. Todo el sistema opera mediante un esquema de ejecución cíclica de tareas no bloqueantes que se actualiza ininterrumpidamente cada 1 milisegundo. Esta restricción arquitectónica es obligatoria para garantizar un comportamiento comunitario del firmware, asegurando que ningún módulo bloqueante se apropie del control de la CPU de forma exclusiva.

El modelo de comportamiento de los módulos en código C temporizado fluye lógicamente a través de estas tres etapas. Durante la fase de escrutinio, el ejecutivo cíclico consulta repetidamente si existe algún cambio de estado en las entradas digitales correspondientes a los sensores. Si detecta una variación física en el entorno, el módulo del sensor deposita inmediatamente un mensaje en la interfaz del sistema.

Posteriormente, en la fase de procesamiento, el módulo central verifica constantemente la llegada de estos nuevos mensajes, los carga en memoria y evalúa si la lógica de control demanda generar un cambio de estado. De ser así, el sistema emite un nuevo mensaje de intrucción apuntando directamente hacia la interfaz de los actuadores. Finalmente, en la etapa de actuación, el ejecutivo revisa si existen mensajes pendientes dirigidos a los actuadores para luego procesarlos y ejecutar físicamente la acción solicitada en las salidas digitales.
